Overview

The goal of this framework is to allow modders to make generic materials for creatures much quicker and easier than the old method of "Create Creature with Material(s), Create new Tissue_Templates to use the generic material(s), create BDPs that use new templates" which if you wanted to do more than one type of generic material (white meat, red meat, reptiles, avians, canines, etc) it's a lot of cloning of templates just to do that.

That is where these Creature Vartiant files come into play, now you don't need to touch too much of vanilla (BDPs, Templates) you just need to setup your creatures that have the materials you want, then override the existing tissues in the creatures you want. Will there still be a lot of find/replace and fine tuning, yes, but now you don't have to worry about creating a whole batch of templates and other things to get it done.

This can also be used on a smaller scale, say you wanted to make Holy dwarves, Demon Dwarves, Hill Dwarves. And you didn't want there to be 3-4 or 50 dwarf materials, you can use this framework to say Holy Dwarves use vanilla DWARF materials, and now you've already cut down on the number of materials, but still keeping the core list of vanilla intact.

  1. <structural_tissue_layer>
  2. <fat_tissue_layer>
  3. <muscle_tissue_layer>
  4. <tissue_id_suffix>
  1. This is normally CHITIN but can be anything you want
  2. This is basically used for FAT layers
  3. This is basically MUSCLE layers

This will add to all tissue ids (even the ones passed in) to be _"SUFFIX" that you pass in, so basically if you used [BODY_DETAIL_PLAN:this-method:CHITIN:FAT:MUSCLE:TOUGH] the inner workings would translate say a line like this: [BP_LAYERS:BY_CATEGORY:BODY:ARG3_ARG4:6:ARG2_ARG4:1:ARG1_ARG4:2] into [BP_LAYERS:BY_CATEGORY:BODY:MUSCLE_TOUGH:6:FAT_TOUGH:1:CHITIN_TOUGH:2] of course all organ or hard references will also get suffixed with _"SUFFIX"

c_variation_0_tissue_generic ADD_TISSUE
  1. <tissue_id>
  2. <tissue_template>
  3. <material>
  1. Can be anything as long as it hasn't been used before example would be SKIN.
  2. This will be the TISSUE_TEMPLATE that this tissue will use.
  3. The material that will override the tissue's material, however as it is a CV you will have to use '|' pipes in place of ':' colons. Example CREATURE_MAT|DOG|LEATHER or INORGANIC|IRON

Filled example:
c_variation_0_tissue_generic ADD_TISSUE_NAME
  1. <tissue_id>
  2. <tissue_template>
  3. <material>
  4. <tissue_name>
  1. Can be anything as long as it hasn't been used before example would be SKIN.
  2. This will be the TISSUE_TEMPLATE that this tissue will use, so to match with 1. you'd use SKIN_TEMPLATE but it can honestly be any other template you want FAT_TEMPLATE, etc.
  3. The material that will override the tissue's material, however as it is a CV you will have to use '|' pipes in place of ':' colons. Example CREATURE_MAT|DOG|LEATHER or INORGANIC|IRON
  4. This will override the tissues name to what is given membrane|NP so like the example from 1. this would create a tissue with the ID of SKIN using the SKIN_TEMPLATE the Material would be IRON but instead of being called "skin" it will be called "membrane".

c_variation_0_tissue_vanilla_clone CHITIN_CREATURE_TISSUES
  1. <creature_id>
  1. This will be the creature that will be used to set the material, it also assumes that the <tissue_id> also matches the material_id, like the above example of FAT in vanilla there is a FAT material, FAT tissue template, and then the FAT tissue that is built from the tissue template and uses the FAT material.
*** This method basically adds all tissues that vanilla's BDP CHITIN_TISSUES adds, only sets the material to that of the picked creature
Filled example:
c_variation_0_tissue_generic COPY_CREATURE_TISSUE
  1. <creature_id>
  2. <tissue_id>
  1. This will be the creature that will be used to set the material, it also assumes that the <tissue_id> also matches the material_id, like the above example of FAT in vanilla there is a FAT material, FAT tissue template, and then the FAT tissue that is built from the tissue template and uses the FAT material.
  2. This method goes off the assumption that the <tissue_id> will also match a "tissue_id_TEMPLATE" Example is FAT this method assumes there is a FAT_TEMPLATE to use, however using something like IRONSKIN there will need to be an IRONSKIN_TEMPLATE which in vanilla does not exist.
*** This method is mostly used in cases where you might want to USE an already familar vanilla material/tissue combo like having a bird use the FEATHER of another bird, so that you don't clutter up the number of materials in game (BIRD has FEATHER material, BIRD_EVIL_VERSION uses the FEATHER material from BIRD for its FEATHER tissue)
Filled example:

c_variation_0_basic EGG_LAYERS_LEATHERY_SIZE
  1. <creature_id>
  2. <is_leathery>
  3. <eggsize>
  1. Creature to take the egg materials from.
  2. This can be only two values, YES or NO. YES will make the eggshell from a EGGSHELL_LEATHERY material, if it is NO it will make the eggshell with a EGGSHELL material. If this is any other value the egg will only have WHITE and YOLK
  3. Size of the egg(s) in cm3

c_variation_0_tissue_modular NAIL_TISSUE_CREATURE
  1. <creature_id>
  2. <nail_type>
  3. <layer_on_fingers>
  4. <layer_on_toes>
  1. Creature to take the materials from.
  2. Nail type to use which is NAIL, CLAW, or TALON.
  3. If this is YES it will then apply this tissue to the FRONT of all bodyparts categorized as FINGER
  4. If this is YES it will then apply this tissue to the FRONT of all bodyparts categorized as TOE
